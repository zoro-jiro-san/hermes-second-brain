## Daydream Session: Memory as a Travelling Wave

### Seed Concept
Cross-domain synthesis connecting **mycorrhizal intelligence** (underground fungal networks), **stigmergic pressure-field coordination** (LLM agents coordinating through shared artifacts), and **Hermes agent architecture**.

### Key Insights

#### 1. Memory Compression ≠ Deletion — It's Fusion
The mycorrhizal insight: hyphae self-regulate density by *merging*. Applied to agent memory: instead of dropping old messages, merge similar ones into generalized patterns that preserve information density while reducing token count.

#### 2. The Wavefront Metaphor Solves Exploration-Exploitation
Current agents either over-rely on existing knowledge or constantly re-derive from scratch. A travelling wave naturally balances: the wavefront explores, the established network exploits, and self-regulating density ensures neither dominates.

#### 3. Pressure Alignment > Prompt Engineering
From the pressure-field paper (Rodriguez, Jan 2026): LLM agents coordinating through shared artifact + pressure gradients achieved **48.5% solve rate vs 1.5% hierarchical**. Agents don't need to communicate intentions — they just need aligned objective functions.

#### 4. The Universal Pattern
All three domains share: **Self-organizing intelligence through local gradient-following on a shared medium, with temporal dynamics that prevent premature convergence.**
- Local gradient-following = each actor only needs local information
- Shared medium = the environment IS the coordination mechanism
- Temporal dynamics = decay prevents getting stuck, but decay is *reallocation* not destruction
- Self-organizing = no central planner needed

### Concrete Architecture Proposal

```
Memory Field Structure:

[Wavefront — last 7 days]
├── Dense, detailed, structured
├── Full context preserved
├── Active in every conversation
└── Raw experiences, verbatim corrections

[Active Transport Zone — 7-60 days]
├── Compressed but structured
├── Patterns extracted from raw experiences
├── Key decisions and outcomes preserved
└── Generalized rules forming

[Established Network — 60+ days]
├── Highly compressed patterns
├── Fused knowledge (merged similar experiences)
├── Transport-efficient: quick to load, broad coverage
└── "Trunk routes" — heavily reinforced pathways
```

### Practical Applications
- **Short-term:** Memory entries with automatic "age + retrieval" scoring
- **Medium-term:** Pressure-gradient conversation history (unresolved questions maintain pressure)
- **Long-term:** Full travelling-wave memory architecture

### Ideas Worth Implementing
1. **PheroPath-style filesystem metadata** — Tools write xattr/sidecar metadata describing what they explored, rejected, and recommend
2. **Memory fusion algorithm** — When entries have >70% topic overlap, propose merged entries
3. **Skill trunk routes** — Detect shared sub-procedures across skills, extract into shared references
