# Skill Linking Strategy — Synthesis Artifacts

**Date:** 2026-04-26  
**Source:** Synthesis of 4 research reports → 4 new skill files  
**Target:** `~/.hermes/skills/` (Hermes Agent skill loading directory)  
**Method:** Symbolic links (symlinks) from synthesis workspace to live skill directory

---

## 1. Overview

This strategy describes how the newly created skill files (generated by the Synthesis Agent) are linked into Hermes Agent's active skill directory. Using **symlinks** rather than copies provides:

- **Single source of truth**: Edits happen in `/home/tokisaki/work/synthesis/skills/`, reflected live in `~/.hermes/skills/`
- **Easy updates**: Re-running synthesis overwrites source files; Hermes automatically sees updates
- **No duplication**: Same inode, two directory entries
- **Version control**: Synthesis workspace can be tracked independently of Hermes config
- **Fast iteration**: Modify skill in synthesis → instantly available to Hermes

## 4. Symlink Creation Commands

Run these commands **as the Hermes user** (typically `tokisaki`):

```bash
# Ensure target categories exist
mkdir -p ~/.hermes/skills/mlops
mkdir -p ~/.hermes/skills/autonomous-ai-agents
mkdir -p ~/.hermes/skills/product-strategy

# Create symlinks (relative paths preferred for portability)
cd ~/.hermes/skills/mlops
ln -sf ../../../../work/synthesis/skills/obscura-ai-obscura obscura

cd ~/.hermes/skills/autonomous-ai-agents
ln -sf ../../../../work/synthesis/skills/chainyo-claude-task-master claude-task-master
ln -sf ../../../../work/synthesis/skills/agentic-stack-agentic-stack agentic-stack

cd ~/.hermes/skills/product-strategy
ln -sf ../../../../work/synthesis/skills/anthropic-claude-ads claude-ads

# Verify
ls -la ~/.hermes/skills/mlops/obscura
ls -la ~/.hermes/skills/autonomous-ai-agents/claude-task-master
ls -la ~/.hermes/skills/autonomous-ai-agents/agentic-stack
ls -la ~/.hermes/skills/product-strategy/claude-ads
```

**What each symlink points to**:
- `~/.hermes/skills/mlops/obscura` → `/home/tokisaki/work/synthesis/skills/obscura-ai-obscura`
- `~/.hermes/skills/autonomous-ai-agents/claude-task-master` → `/home/tokisaki/work/synthesis/skills/chainyo-claude-task-master`
- `~/.hermes/skills/autonomous-ai-agents/agentic-stack` → `/home/tokisaki/work/synthesis/skills/agentic-stack-agentic-stack`
- `~/.hermes/skills/product-strategy/claude-ads` → `/home/tokisaki/work/synthesis/skills/anthropic-claude-ads`

**Note**: The skill directory name (symlink name) should match the `name:` field in SKILL.md frontmatter for consistency.

## 5. Loading & Discovery

Hermes Agent automatically loads all skills from subdirectories of `~/.hermes/skills/` that contain a `SKILL.md` file. No additional registration is needed once the symlinks are in place.

To verify installation:

```bash
# List all loaded skills
hermes skills list

# Check a specific skill
hermes skills info obscura

# Test loading
hermes chat -q "Generate synthetic training data using Obscura" --skills obscura
```

## 6. Update Workflow

When synthesis re-runs (new research or refreshed information):

1. Synthesis Agent overwrites source `SKILL.md` files at `/home/tokisaki/work/synthesis/skills/<repo>/`
2. Symlinks automatically point to updated content (no action needed)
3. Optionally run `hermes skills reload` to refresh in-memory cache (if Hermes caches skill content)
4. Users get updated knowledge immediately

**Do not** edit files directly under `~/.hermes/skills/` — always edit the synthesis source, then symlink will propagate.

## 7. Backup & Version Control

**Synthesis workspace** (`/home/tokisaki/work/synthesis/`) should be committed to a Git repository independent of Hermes's own repo. This allows:

- Tracking skill evolution over time
- Roll back to previous skill versions if issues arise
- Share skill bundles across Hermes deployments
- CI/CD validation of skill file format

**Hermes config** (`~/.hermes/`) typically stays out of VCS (user-specific). Symlinks are cheap to recreate.

## 8. Troubleshooting

| Symptom | Diagnosis | Fix |
|---------|-----------|-----|
| Skill not loading | SKILL.md missing or malformed frontmatter | `cat ~/.hermes/skills/<skill>/SKILL.md`; check `---` delimiters |
| "Skill not found" error | Symlink broken or points to wrong location | `ls -la ~/.hermes/skills/<category>/<skill>`; verify target exists |
| Stale content | Hermes cached skill at startup | Restart Hermes or run `hermes skills reload` |
| Permission denied | Hermes process user lacks read access to synthesis dir | Ensure `chmod -R o+rX /home/tokisaki/work/synthesis/skills/` or run Hermes as same user |
| Duplicate skill names | Two skills with same `name:` in frontmatter | Rename one; SKILL.md `name` must be globally unique |
| Broken symlink loop | Relative path miscalculated | Use absolute paths for reliability: `ln -sf /home/tokisaki/work/synthesis/skills/... <target>` |
